Detailed Description

The GameObject abstract class represents an in-game entity composed of one or more components. By delegating functionality to components, GameObject enables flexible and extensible behavior without requiring complex inheritance structures.

Clean up the GameObject and all of its attached components.

This method should be called when the GameObject is no longer needed, such as when transitioning to a new game state or shutting down the engine. It ensures that each component releases its resources and that the component list is cleared.

Retrieve a component of the specified type from this GameObject.

This templated method returns the first attached component that matches the requested type T, which must implement IComponent. If no matching component is found, it returns null.

Example:
