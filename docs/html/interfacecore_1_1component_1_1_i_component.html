Detailed Description

The IComponent interface defines a contract for components that can be attached to a GameObject. Components allow behavior to be encapsulated and reused across different entities in the game. Typical examples include components for rendering, physics, AI, input handling, audio playback, or any other subsystem that affects the game object's state or presentation.

void core::component::IComponent::cleanup ( )

Perform cleanup tasks for this component. This method should be called when the component is removed from its GameObject or when the GameObject itself is being destroyed. Implementations should handle releasing any resources, removing event listeners, or performing other cleanup tasks to prevent memory leaks and ensure the component can be safely removed.

◆ update()

void core::component::IComponent::update ( GameObject obj)

Update the state of this component. This method is typically called every frame or update tick. Within this function, the component should read and modify the GameObject's state as necessary, respond to events, and perform any per-frame logic (e.g., moving objects, playing animations, responding to user input).

Params: obj = The GameObject to which this component is attached. The component may use this reference to query and modify the object's properties (e.g., position, rotation), or to access other attached components.
